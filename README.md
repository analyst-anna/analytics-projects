## О проекте

**Заказчик:** ИП ... (продажа галантереи на Wildberries)  
**Метод поставки:** FBS (товар на складе WB)  
**Бизнес-задача:** Создать автоматизированный инструмент для оценки реальной рентабельности бизнеса

### Ключевые боли бизнеса

| Боль | Решение в дашборде |
|------|---------------------|
|  Не понимают реальные затраты (логистика, комиссия, налоги) | Расчёт валовой прибыли с учётом всех затрат |
|  Сложно оценить, какой товар перестают покупать | Топы лидеров роста и падения с динамическим периодом |
|  Нет понимания приоритетных складов | Географический анализ продаж по областям |

### Что сделано

- Дашборд в Power BI с подключением к Postgres (API WB) и Google Sheets
- Расчёт валовой прибыли с учётом возвратов, логистики и комиссий
- Автоматические топы лидеров роста и падения
- Географический анализ продаж для складской оптимизации
- 13 DAX-мер для всех метрик из ТЗ

## 🔑 Бизнес-инсайты

### ✅ Выявлено в ходе анализа:

1. **Высокая рентабельность** — 59% указывает на эффективную бизнес-модель
   - Валовая прибыль составляет более половины выручки
   - Бизнес генерирует стабильный денежный поток

2. **Географическая концентрация** — основная выручка приходится на Москву и МО
   - Центральный федеральный округ доминирует в продажах
   - Есть потенциал для расширения в другие регионы

3. **Неравномерная динамика** — наблюдаются сезонные пики и спады
   - Периоды высокой активности чередуются с низкими продажами
   - Требуется планирование запасов с учётом сезонности

4. **Товарная дифференциация** — некоторые артикулы показывают рост >1000%, другие падают
   - Поляризация ассортимента по эффективности
   - Необходима оптимизация товарной матрицы

## 💡 Рекомендации

### 1. Оптимизация логистики
**Пересмотреть приоритетные склады с учётом географии продаж**

**Действия:**
- Увеличить запасы на складах Москвы и МО (основной спрос)
- Проанализировать целесообразность складов в регионах с низким спросом
- Оптимизировать маршруты доставки для снижения логистических затрат

**Ожидаемый эффект:**
- Снижение затрат на логистику на 10-15%
- Ускорение доставки до клиентов

---

### 2. Ассортиментная политика
**Сфокусироваться на товарах-лидерах роста**

**Действия:**
- Увеличить закупки артикулов с ростом >500%
- Вывести из ассортимента товары с устойчивым падением
- Провести ABC-анализ ассортимента

**Ожидаемый эффект:**
- Рост оборачиваемости товаров
- Увеличение общей выручки на 15-20%

---

### 3. Работа с возвратами
**Проанализировать причины возвратов топ-артикулов**

**Действия:**
- Выявить топ-10 товаров по количеству возвратов
- Проанализировать причины (брак, несоответствие описанию, размер)
- Улучшить карточки товаров (фото, описание, размерная сетка)
- Ввести контроль качества перед отгрузкой

**Ожидаемый эффект:**
- Снижение процента возвратов на 20-30%
- Экономия на логистике возвратов
- Повышение рейтинга продавца на WB

---

### 4. Ценообразование
**Учесть реальную себестоимость с логистикой при установке цен**

**Действия:**
- Рассчитать полную себестоимость для каждого артикула:
  - Закупочная цена
  - Комиссия WB (12%)
  - Логистика (55-220 ₽/ед. в зависимости от категории)
  - Хранение (0.08 ₽/день)
  - Налог (2% от выручки)
- Установить минимальную цену с целевой рентабельностью 40-50%
- Внедрить динамическое ценообразование для товаров с низкой маржой

**Ожидаемый эффект:**
- Исключение убыточных продаж
- Стабилизация рентабельности на уровне 50-60%

## 🧮 Реализованные DAX-меры

### 1. Валовая прибыль (с учётом возвратов и логистики)

```dax
Валовая прибыль = 
SUMX (
    'Номенклатура',
    VAR SalesRevenue =
        CALCULATE (
            SUM ( 'Реализация'[retail_amount] ),
            'Реализация'[doc_type_name] = "Продажа"
        )
    VAR ReturnRevenue =
        CALCULATE (
            SUM ( 'Реализация'[retail_amount] ),
            'Реализация'[doc_type_name] = "Возврат"
        )
    VAR NetRevenue = SalesRevenue - ReturnRevenue
    
    VAR SalesCommission =
        CALCULATE (
            SUM ( 'Реализация'[ppvz_sales_commission] ),
            'Реализация'[doc_type_name] = "Продажа"
        )
    VAR ReturnCommission =
        CALCULATE (
            SUM ( 'Реализация'[ppvz_sales_commission] ),
            'Реализация'[doc_type_name] = "Возврат"
        )
    VAR NetCommission = SalesCommission - ReturnCommission
    
    VAR QtySold =
        CALCULATE (
            SUM ( 'Реализация'[quantity] ),
            'Реализация'[doc_type_name] = "Продажа"
        )
    VAR QtyReturned =
        CALCULATE (
            SUM ( 'Реализация'[quantity] ),
            'Реализация'[doc_type_name] = "Возврат"
        )
    VAR NetQty = QtySold - QtyReturned
    
    VAR LogisticsPerUnit = 
        LOOKUPVALUE (
            'Тарифы'[Cтоимость логистики],
            'Тарифы'[subject], 'Номенклатура'[subject]
        )
    VAR NetLogistics = NetQty * LogisticsPerUnit
    
    VAR CostPerUnit = 
        LOOKUPVALUE (
            'Себестоимость'[Себестоимость],
            'Себестоимость'[Штрих-код], 'Номенклатура'[barcode]
        ) / 100   -- Делим на 100, т.к. себестоимость завышена в 100 раз
    
    VAR NetCost = NetQty * CostPerUnit
    
    RETURN
        NetRevenue - NetCommission - NetLogistics - NetCost
)

## 2. ROI

```dax
ROI = 
VAR GrossProfit = [Валовая прибыль]

VAR Investment = 
    SUMX(
        'Реализация',
        VAR NetQty = 
            SWITCH(
                TRUE(),
                'Реализация'[doc_type_name] = "Продажа", 'Реализация'[quantity],
                'Реализация'[doc_type_name] = "Возврат", -ABS('Реализация'[quantity]),
                0
            )
        VAR Barcode = 'Реализация'[barcode]
        VAR CostPerUnit = 
            CALCULATE(
                AVERAGE('Себестоимость'[Себестоимость]),
                FILTER('Себестоимость', 'Себестоимость'[Штрих-код] = Barcode)
            )
        RETURN
            NetQty * CostPerUnit
    )

RETURN
    DIVIDE(GrossProfit, Investment, 0)
## 3. Лидеры роста (динамическое окно)

```dax
Изменение_выручки = 
VAR CurrentValue = [Выручка WB]
VAR PreviousValue = [Выручка_пред_период]
RETURN
    IF(
        NOT ISBLANK(PreviousValue),
        CurrentValue - PreviousValue,
        BLANK()
    )

Изменение_выручки_% = 
DIVIDE(
    [Изменение_выручки],
    [Выручка_пред_период]
)
